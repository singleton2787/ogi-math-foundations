# Contributing

This is a research repository. Contributions are welcome in specific areas.

---

## What We Accept

- **Bug fixes** in benchmark code (`experiments/`)
- **Reproducibility improvements** — better seeding, cleaner output, cross-platform fixes
- **Typos and clarifications** in the README or ROADMAP
- **New experiments** that test claims made in the paper (open an issue first to discuss)

## What We Do Not Accept

- Changes to the core proof logic without a corresponding issue discussion
- New features unrelated to the paper's scope
- Dependencies that break CPU-only reproducibility

---

## Process

1. Open an issue describing what you want to fix or add
2. Fork the repo and make your changes on a branch
3. Submit a PR with a clear description of what changed and why
4. One of the authors will review

---

## Code Style

- Python 3.10+
- `black` for formatting
- Docstrings on all public functions
- No new dependencies without discussion
